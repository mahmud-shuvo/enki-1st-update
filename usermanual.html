Focus and navigation

Few tips to minimize mouse usage.
You can jump to any dock widget by pressing its shortcut. Example:

  • Alt+F - show File browser, move focus to it

Press any search or replace shortcut (Ctrl+F, Shift+Ctrl+R, ...) to jump to search dialogue and set focus to Search line Esc closes focused widget. Example:

  • Alt+F,Esc - close File browser

Shift+Esc closes all visible docks and search dialogue.
Ctrl+Enter moves focus to current document

Files

Create file

If mksv3 is executed from console with file names, it creates not existing files.

Open file

Open file dialog usually opened in directory, where current file located. If you want to open file from some directory, sometimes it's quicker to select in the opened file list file from this directory, and only then press Ctrl+O or Ctrl+L

Sort opened file list

By default documents are sorted by URI. If you want to change sort order (i.e. sort by opening order or file name), rightclick a list item and go to Sorting menu. You also can sort documents manually, by dran-n-dropping items in the list

Locator

mksv3 supports classic Open File dialog, which you see when press Ctrl+O. But, it is terrible. With Locator (Ctrl+L) you can open files much quicker.

File Browser

File browser supports few shortcuts, which may be used, when tree is focused:

Backspace Move one level up
~ (actually, `) Move to home directory
. (dot) Move to directory, in which current file is located

Advanced replacements

mksv3 search functionality supports replacements with escape sequences and groups from regular expression. To insert \n or other sequence, just enter it to the Replace field. Example:

replace text: "I want to break\nthis line"
result: "I want to breakb
this line"

List of supported sequences

If you don't want mksv3 replace your \something sequences, escape it with additional slash: \\n. Example:

replace text: "I don't want to break\\nthis line"
result: "I don't want to break\nthis line"

Groups from regular expression match are referenced as \{number}. \1 - first submatch, \2 - second... Example:

text: "foo bar"
pattern: "((\w+) (\w+))"
replace text: "\1 replaced with \3 \2"
result: "foo bar replaced with bar foo"
